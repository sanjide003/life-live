# Livelife Privacy Policy Draft

Livelife is designed as a local-first personal life operating system.

## Data stored locally

Tasks, habits, goals, finance entries, health entries, prayer records, notes, reviews, settings, reminders, and reports are stored on the user's device by default.

## Optional cloud backup

Firebase backup is optional. Data is uploaded only when the user signs in and enables Firebase backup.

## Health data

Health Connect / Google Fit permissions are requested only when the user taps Connect. Water, mood, and medicine remain manual entries.

## Notifications

Notifications remain off until Android notification permission is granted.

## Excluded sensitive data

The first version does not include a password vault, ID document storage, SMS parsing, or location history.

## Data deletion

Users can disable backup, export local data, import backup files, and request local/cloud data deletion from Settings.
